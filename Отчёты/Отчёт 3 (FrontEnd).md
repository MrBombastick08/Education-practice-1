# Отчет по Front-end части проекта 

## 1. Введение

Данный отчет описывает Front-end часть веб-приложения, разработанного на базе фреймворка Flask. Front-end отвечает за представление данных пользователю, обеспечение навигации и обработку пользовательского ввода через формы.

## 2. Технологический стек

Front-end проекта использует стандартный набор веб-технологий, интегрированных с шаблонизатором Jinja2, который является частью Flask:

| Технология | Назначение |
| :--- | :--- |
| **HTML5** | Структура и семантика веб-страниц. |
| **CSS3** | Стилизация и оформление пользовательского интерфейса. |
| **JavaScript (Vanilla JS)** | Клиентская логика, в основном для улучшения UX (например, подтверждение удаления, адаптация бокового меню). |

## 3. Структура Front-end

Front-end организован в соответствии с требованиями Flask, используя две основные директории: `templates` для HTML-шаблонов и `static` для статических ресурсов.

### 3.1. Директория `templates` (Шаблоны)

Шаблоны используют принцип наследования для обеспечения единообразия дизайна и упрощения поддержки.

| Файл шаблона | Назначение |
| :--- | :--- |
| **`base.html`** | **Базовый макет.** Содержит общую структуру страницы (Header, Sidebar, Footer), подключение стилей и скриптов, а также логику отображения flash-сообщений. Все остальные шаблоны наследуют его. |
| `index.html` | Главная страница приложения. |
| `products.html` | Страница со списком всей продукции. |
| `product_form.html` | Шаблон для добавления и редактирования продукции (использует формы Flask-WTF). |
| `material_calculator.html` | Страница калькулятора сырья. |
| `workshops.html` | Страница с детальной информацией о цехах, задействованных в производстве конкретного изделия. |

### 3.2. Директория `static` (Статические ресурсы)

| Файл/Директория | Назначение |
| :--- | :--- |
| `static/css/style.css` | **Основной файл стилей.** Содержит все CSS-правила для оформления приложения. |
| `static/js/main.js` | **Основной файл скриптов.** Содержит клиентскую логику (например, функции для адаптации бокового меню и подтверждения удаления). |
| `static/img/` | Хранение изображений (логотип, фавикон). |

## 4. Дизайн и Пользовательский Интерфейс (UI/UX)

### 4.1. Дизайн

Дизайн приложения характеризуется следующими особенностями:

*   **Цветовая схема:** Используется корпоративная цветовая палитра, где основным цветом является темно-синий (`#355CBD`), ассоциирующийся с надежностью и профессионализмом.
*   **Шрифты:** Используется шрифт 'Candara' (или его запасные варианты), обеспечивающий хорошую читаемость.
*   **Макет:** Приложение использует двухколоночный макет с фиксированным верхним **Header** и постоянным боковым **Sidebar** для навигации. Основной контент отображается в центральной области (`.content`).

### 4.2. Пользовательский Опыт (UX)

*   **Навигация:** Реализована через два элемента: верхнее меню (`.nav-menu`) и боковое меню (`.sidebar-menu`). Боковое меню использует логику подсветки активного пункта (`class="active"`) для улучшения ориентации пользователя.
*   **Адаптивность:** В файле `style.css` присутствуют медиа-запросы (`@media (max-width: 576px)`), что указывает на попытку сделать интерфейс адаптивным для мобильных устройств, переводя боковое меню в линейный вид.
*   **Клиентская логика:** В `base.html` и `main.js` реализованы важные UX-функции:
    *   **Подтверждение удаления:** Перед отправкой формы удаления пользователю показывается диалоговое окно с подтверждением (`confirm('Вы уверены...')`).
    *   **Flash-сообщения:** Уведомления об успешных операциях (`alert-success`) или ошибках (`alert-danger`) отображаются в верхней части контента.

## 5. Вывод

Front-end проекта является функциональным и интуитивно понятным. Он эффективно использует возможности шаблонизатора Jinja2 для динамического отображения данных, полученных от Back-end. Дизайн чистый и профессиональный, а наличие базовой адаптивности и клиентской логики (подтверждение удаления, подсветка меню) улучшает общее впечатление от использования приложения.
